The Problem

Every organisation has the same shape of pain.

  • Multiple source systems — different ERPs, different column names, different export formats. No canonical truth.
  • Reconciliation is manual — spreadsheets, ad-hoc queries, tribal knowledge. Findings don't accumulate.
  • Domain expertise is scarce — the person who knows what “normal” looks like retires, changes roles, or was never hired.
  • Every new domain starts from zero — new industry, new consultants, new six-month discovery phase.

Universal Patterns

The diagnostic shapes are the same everywhere

  • Balance probes — compare two aggregates. Usage vs billing. Received vs sold. Ordered vs delivered.
  • Duplicate detection — same shape everywhere. Duplicate invoices, shipments, purchase orders.
  • Trend analysis — rolling average drift. Costs, lead times, defect rates. The math is identical.
  • Taxonomy hierarchies — cost centres, product categories, regional structures. Same closure table, different labels.

The entity names change. The architecture doesn't.
